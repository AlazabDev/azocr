# خطة الاختبار الأولي (Smoke Test)

دليل سريع لتشغيل أول اختبار تشغيلي للتأكد من أن واجهة الـMVP ومسارات الـAPI الأساسية تعمل قبل دمج تكاملات السحاب.

## المتطلبات المسبقة
- Node.js 18 أو أحدث
- npm 9 أو أحدث
- حساب Google Cloud/ElasticSearch/Google Drive (اختياري للتجارب الحقيقية)

## تجهيز البيئة
1) انسخ ملف المتغيرات:
```bash
cp .env.local.example .env.local
```
2) حدّث قيم الاعتماد حسب حساباتك عند توفرها. يمكن ترك القيم الافتراضية للتجربة المحلية لأن المسارات تحتوي Fallback.
3) ثبّت الحزم:
```bash
npm install
```

## خطوات الاختبار السريع
1) التحقق الساكن:
```bash
npm run verify
```
- يشمل ESLint + فحص TypeScript بلا إخراج.

2) تشغيل التطوير:
```bash
npm run dev
```
- افتح `http://localhost:3000`.
- جرّب رفع ملف PDF/Excel صغير ولاحظ البطاقات (حصر الكميات/تحليل الأسعار/مقارنة العروض/لوحة التقارير).

3) تجربة التكاملات الاختيارية:
- **OCR السحابي**: اضغط زر "تشغيل Vision OCR" بعد رفع ملف. لو لم يكن هناك بيانات اعتماد، ستظهر بيانات Fallback محلية.
- **ElasticSearch**: اكتب في حقل البحث الفوري. في حال عدم تفعيل ElasticSearch ستظهر نتائج مفلترة محلياً.
- **Google Drive**: افتح بطاقة المزامنة. ستظهر عينات إذا لم تتوافر بيانات خدمة Drive.

4) بناء الإنتاج (اختبار جاهزية النشر):
```bash
npm run build
```

## ملاحظات سجل الاختبار
- دوّن أي أخطاء تظهر في الطرفية أو الشبكة مع لقطات شاشة/سجلات API.
- إذا فشل البناء بسبب اعتمادات ناقصة، علق الأسطر الخاصة بالاعتماد أو وفّر القيم الصحيحة في `.env.local` وأعد المحاولة.
